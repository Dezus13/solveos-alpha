# User Stories

## Product goal

These stories describe the people SolveOS is designed for and the decisions they need help making.

## Core features

- Founder decision support.
- Pricing and market-entry analysis.
- Risk discovery through a Skeptic advisor.
- Clear verdicts with next steps.
- Memory and outcome tracking after decisions are made.

## User flow

1. User explains a real decision scenario.
2. SolveOS asks for missing details like budget, timeline, or constraints.
3. AI advisors analyze the decision.
4. User receives a blueprint with recommendation and risks.
5. User returns later to record the outcome.

## Stories

### Story 1: Founder starting a company

As a founder with a stable job, I want to compare staying employed versus quitting to start a company, so I can understand opportunity cost, market timing, and personal risk.

### Story 2: Founder choosing pricing

As a founder with beta users, I want to compare flat pricing and usage-based pricing, so I can choose the model with stronger long-term value.

### Story 3: Team choosing a market

As a small team with limited resources, I want to compare entering the US market versus focusing on Europe first, so I can choose the best rollout path.

## Technical overview

Each story should become a test scenario for the Decision Engine. The expected output should include a clear verdict, risks, confidence, and action items.

## Roadmap / order of work

1. Use these stories as sample inputs.
2. Check whether the blueprint answers each user goal.
3. Add missing fields to the blueprint format.
4. Use outcomes from real users to improve future stories.
