# Spec Consistency Rules

All specs must describe the same system.

Rules:
- No contradictions between files
- Same terms must be used everywhere
- Use: Decision, Action, UserState
- Use: Decision Journal, Identity Engine, Pressure Layer
- User Flow must match Action System
- Decision Engine must match User Flow
- Identity Engine and Pressure Layer must match Action outcomes

If something changes:
- Update ALL related specs
- Do not leave outdated logic
