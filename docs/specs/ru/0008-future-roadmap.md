# 0008 Дорожная карта (Future Roadmap)

> **Краткое резюме модуля**
> - **Что делает:** определяет следующие технические и продуктовые улучшения, сфокусированные на петле Решение → Действие
> - **Зачем нужен:** даёт команде общее понимание приоритетов MVP и направлений развития после валидации
> - **Что НЕ должен делать:** не описывает текущую реализацию — только будущие шаги
> - **Конфликты:** нет; это перспективный документ, не описывающий текущее поведение системы

---

## 1. Назначение

- Определить следующие технические и продуктовые улучшения.
- Сохранить будущую работу сфокусированной на петле Решения и Действия.

## 2. Где используется

- Планирование проекта.
- Обзор с ментором.
- Будущая работа по реализации.
- Валидация MVP.

## 3. Основные объекты

- **UserAccount:** будущий владелец каждого Решения.
- **Database:** будущее место сохранения Решений и Действий.
- **Decision Journal (Журнал решений):** будущий вид долгосрочной истории.
- **PaymentSignal:** доказательство готовности пользователей платить.
- **FeedbackItem:** реальная обратная связь от пользователей.
- **TestCoverage:** проверки основных потоков.

## 4. Логика (пошагово)

1. Валидировать потребность пользователя.
2. Собрать обратную связь от пользователей.
3. Протестировать готовность платить.
4. Улучшить хранилище.
5. Добавить учётные записи пользователей.
6. Улучшить тесты.
7. Улучшить поток обзора.

## 5. Хранимые данные

- `userId`: будущий владелец Решений.
- `paymentInterest`: готовность пользователя платить.
- `feedbackText`: обратная связь от пользователя.
- `decisionHistory`: сохранённые Решения в базе данных.
- `outcomeHistory`: сохранённые результаты с течением времени.

## 6. Граничные случаи

- Пользователи не платят: пересмотреть предложение продукта.
- Обратная связь неясна: провести более мелкие тесты.
- Миграция базы данных не удаётся: сохранить локальную резервную копию.
- Учётные записи добавляют трение: сохранить простой демо-режим.
- Тесты не проходят: блокировать релиз до исправления.
