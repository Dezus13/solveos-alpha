# 0006 Данные и хранилище (Data And Storage)

> **Краткое резюме модуля**
> - **Что делает:** определяет, что SolveOS сохраняет, где хранит, как это используется историей и профилем
> - **Зачем нужен:** единый источник истины по всем localStorage-ключам, схемам данных и поведению при граничных случаях хранилища
> - **Что НЕ должен делать:** не описывает логику UI, бизнес-правила давления или AI-промпты
> - **Конфликты:** должен согласовываться с 0004 (Action System) по схеме ActionReminder и 0005 (Identity Engine) по схеме UserState

---

## 1. Назначение

- Определить, что SolveOS сохраняет.
- Поддерживать историю Решений, напоминатели Действий, исходы и UserState.

## 2. Где используется

- Memory API.
- Outcome API.
- Журнал решений (Decision Journal).
- Баннер действия.
- Настройки браузера и состояние профиля.

## 3. Основные объекты

- **Decision (Решение):** сохранённая ситуация пользователя и Вердикт.
- **Action (Действие):** сохранённый следующий шаг.
- **ActionReminder:** сохранённое состояние Действия, включая поля давления.
- **UserState:** сохранённое состояние поведения.
- **Decision Journal (Журнал решений):** сохранённый список прошлых Решений.
- **Outcome:** зафиксированный результат прошлого Решения.
- **ProductSettings:** локальный объект UI-предпочтений.
- **ConversationThread:** локальные сообщения чата, используемые только для непрерывности в текущем браузере.

## 4. Логика (пошагово)

1. Создаётся Решение.
2. Система сохраняет Решение.
3. Система сохраняет ActionReminder с `createdAt`, `dueAt` и `status: not yet`.
4. Журнал решений считывает историю Решений.
5. Пользователь фиксирует исход.
6. Система обновляет UserState.
7. Будущие Решения могут использовать историю.
8. Сообщения чата хранятся локально, и ограниченное окно недавних сообщений отправляется с будущими запросами solve.

## 5. Хранимые данные

### ActionReminder (`solveos_action_pressure_v1`)

- `id`: уникальный идентификатор записи (ключ в объекте хранилища).
- `action`: текст текущего Действия (может быть заменён меньшим действием).
- `decisionText`: исходный текст Решения, породившего Действие.
- `status`: `not yet`, `done`, `skipped` или `overdue`. Устаревшие значения `pending` и `blocked` читаются как действия не-yet до наступления срока.
- `createdAt`: время создания напоминателя. Сбрасывается при принятии меньшего действия.
- `dueAt`: 24-часовой дедлайн. Сбрасывается при принятии меньшего действия.
- `completedAt`: время выполнения Действия.
- `skippedAt`: время пропуска Действия.
- `updatedAt`: время последней записи.
- `blockerCategory`: выбранная категория блокера (`fear`, `unclear`, `lazy`, `external`).
- `smallerAction`: текст уменьшенного действия, сохранённый после выбора `blockerCategory`.
- `overdueScorePenaltyApplied`: булев флаг, устанавливаемый в `true` после применения штрафа −10 за просрочку. Предотвращает двойное списание.

### ActionHistory (производная, не отдельное хранилище)

История вычисляется из `solveos_action_pressure_v1` при считывании:
- Каждый ActionReminder является записью истории, включая активные действия `not yet`.
- Каждая строка показывает текст Решения, текст Действия/результата, нормализованный статус (`done`, `not yet`, `skipped`, `overdue`) и прошедшее время от соответствующей временно́й метки.
- `getHistoryRecords()` возвращает записи, отсортированные по метке результата от новых к старым.
- `getActionMetrics()` возвращает `successRate` (% выполненных в последних 7 записях) и `streak` (последовательность выполненных от новых к старым).
- История действий остаётся внутренней памятью поведения на данный момент — не рендерится в боковой панели до тех пор, пока UI не станет чище.

### Decision (`solveos_saved_decisions`)

- `id`: уникальный идентификатор записи.
- `problem`: Решение пользователя.
- `verdict`: результат Решения.
- `confidence`: оценка Решения.
- `keyRisks`: сохранённые риски.
- `actionText`: следующее Действие.
- `outcomeStatus`: состояние результата.
- `reviewCheckpoints`: контрольные точки на 30, 60 и 90 дней.
- `userDecisionScore`: оценка поведения.

### Pressure State (производное, не хранится)

- `pressure_state`: вычисляется при считывании из прошедших часов `createdAt` (0–2ч = normal, 2–12ч = pressure_2h, 12–24ч = pressure_12h, 24ч+ = overdue).
- `last_action_time`: `createdAt` активного ActionReminder.
- `reset_time`: `createdAt` после вызова `restartWithSmallerAction`.

### User Profile (`solveos_user_profile`)

- `userDecisionScore`: оценка поведения (0–100). Начинается с 50. Обновляется при каждом событии Действия (+5 выполнено, −5 пропущено, −10 просрочено). Ограничена 0–100.
- `decisionScoreTrend`: направление последнего изменения оценки (`up` или `down`).
- `updatedAt`: ISO-метка времени последней записи профиля.
- `riskTolerance`: число от 0 до 1, обновляемое исходами решений.
- `executionScore`: число от 0 до 1, обновляемое исходами решений.
- `biasPatterns`: массив обнаруженных поведенческих паттернов.
- `totalDecisions`: количество отслеживаемых решений.

`userDecisionScore` управляет `getIdentityLabel(score)`, который возвращает одну из четырёх конфронтационных меток:
- 0–39: «You avoid decisions»
- 40–69: «You are inconsistent»
- 70–89: «You act on decisions»
- 90–100: «You are highly reliable»

Оценка показывается в боковой панели через `components/IdentityWidget.tsx`. Обновления передаются в реальном времени через `PROFILE_UPDATED_EVENT`.

### Appearance Settings (`solveos_settings`)

- Управляется `lib/settingsStore.ts`.
- Хранит только глобальные настройки внешнего вида: `theme`, `accent` и `density`.
- `getSettings()`: безопасно считывает настройки на клиенте и использует значения по умолчанию на сервере.
- `saveSettings()`: сохраняет полный объект настроек внешнего вида, применяет его к документу и отправляет `SETTINGS_UPDATED`.
- `updateSettings()`: вносит частичное обновление в текущие настройки.
- `applySettingsToDocument()`: устанавливает `data-theme`, `data-accent` и `data-density` на `document.documentElement`.
- Значения по умолчанию: `theme: system`, `accent: purple`, `density: balanced`.
- `theme`: `system`, `dark` или `midnight`.
- `accent`: `purple`, `blue`, `emerald` или `rose`.
- `density`: `compact`, `balanced` или `calm`.

### ProductSettings (`solveos.productSettings.v1`)

- Хранится в localStorage через `lib/settings.ts` для продуктовых предпочтений, не связанных с внешним видом.
- Значения внешнего вида делегируются в `solveos_settings`, чтобы тема, акцент и плотность имели единый источник истины.
- Невалидные или отсутствующие настройки возвращаются к `defaultSettings`.

### ConversationThread (`solveos.conversation.v1`)

- Хранится в `localStorage` браузера из `components/HomeExperience.tsx`.
- Содержит только недавние ходы чата: `id`, `role`, `content`, `isError` и `timestamp`.
- Максимум хранимых ходов: 60.
- Максимум ходов, отправляемых в `/api/solve`: 12, без ходов с ошибками.
- Используется для:
  - непрерывности уточнений (`«why?»`, `«почему?»`, `«а если нет денег?»`)
  - предотвращения повторяющихся советов
  - оценки сессионного давления
  - контекстного продолжения с сохранением языка
- Очищается минимальным элементом управления «Новый чат» и потоками удаления истории/сброса.
- Это не Журнал решений и не заменяет сохранённые записи Решений.

## 6. Граничные случаи

- Хранилище файлов недоступно: использовать удалённое хранилище при наличии конфигурации.
- Удалённое хранилище отсутствует: использовать локальный файл в разработке.
- Хранилище браузера отсутствует: вернуть пустое состояние.
- Невалидный JSON: игнорировать плохие данные и продолжить.
- Невалидный JSON ConversationThread: игнорировать и начать чистый чат.
- ConversationThread слишком длинный: хранить локально последние 60 ходов и отправлять только последние 12 ходов в API solve.
- Решение не найдено: вернуть безопасную ошибку API.
- localStorage настроек недоступен: использовать значения по умолчанию и сохранить работоспособность приложения.
