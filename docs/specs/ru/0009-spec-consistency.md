# Правила согласованности спецификаций (Spec Consistency Rules)

> **Краткое резюме модуля**
> - **Что делает:** устанавливает правила, гарантирующие, что все спецификации описывают одну и ту же систему без противоречий
> - **Зачем нужен:** предотвращает дрейф между спецификациями — если термины расходятся, код и документация начинают описывать разные системы
> - **Что НЕ должен делать:** не описывает логику системы — только правила согласованности документации
> - **Конфликты:** нет; это мета-документ, управляющий всеми остальными спецификациями

---

Все спецификации должны описывать одну и ту же систему.

Правила:
- Никаких противоречий между файлами
- Одни и те же термины должны использоваться везде
- Использовать: Decision, Action, UserState
- Использовать: Decision Journal, Identity Engine, Pressure Layer
- User Flow должен совпадать с Action System
- Decision Engine должен совпадать с User Flow
- Identity Engine и Pressure Layer должны совпадать с исходами Action

Если что-то меняется:
- Обновить ВСЕ связанные спецификации
- Не оставлять устаревшую логику
