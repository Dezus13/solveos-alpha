# 0008 Монетизация (Monetization)

> **Краткое резюме модуля**
> - **Что делает:** определяет фейковый пэйволл для тестирования готовности платить — без реального платежа, только измерение намерения
> - **Зачем нужен:** валидировать, есть ли у продукта платящая аудитория, прежде чем строить реальную платёжную инфраструктуру
> - **Что НЕ должен делать:** не обрабатывает реальные платежи, не хранит платёжные данные и не связан со Stripe
> - **Конфликты:** должен согласовываться с 0001 (Project Overview) по логике DecisionGate и 0003 (Decision Engine) по тому, что включается в ответ

---

## 1. Назначение

- Протестировать готовность платить до построения реальной платёжной инфраструктуры.
- Скрыть финальный вердикт и следующий шаг за фейковой разблокировкой.
- Измерить: нажимает ли пользователь «Unlock decision — €5»?

## 2. Где используется

- Каждый ход ассистента, содержащий blueprint.
- Показывается в области чата после стриминга частичного ответа с давлением.
- Никакой новой страницы. Никакого Stripe. Никакого чекаута.

## 3. Основные объекты

- **FullResponse:** стримируемый текст, показываемый до ворот. Содержит вердикт, обоснование и следующее действие. Полный ответ показывается свободно.
- **DecisionGate:** UI-компонент, показываемый после полного ответа. До разблокировки: пэйволл более глубокого анализа. После разблокировки: скрытая карта мотивации + анализ рисков + 24-часовой трекер обязательств.
- **UnlockState:** локальное React-состояние (`isUnlocked: boolean`) для каждого хода. Сбрасывается при перезагрузке страницы.
- **PaymentSignal:** нажатие «Unlock full blueprint — €5» считается положительным сигналом.

## 4. Поток (пошагово)

1. Пользователь отправляет Решение.
2. API возвращает полный blueprint (без изменений).
3. `buildAssistantAnswer` создаёт полный полезный ответ:
   - Вердикт (из `blueprint.recommendation`)
   - Обоснование (из `blueprint.diagnosis.coreProblem`)
   - Следующее действие (из `blueprint.actionPlan.today` или `forcedAction`)
4. Полный ответ стримируется как сообщение ассистента.
5. После завершения стриминга `DecisionGate` рендерится ниже сообщения.
6. Состояние до разблокировки показывает:
   - «Want to see what's actually driving this?»
   - «Unlock the hidden motivation map and 24h commitment tracker.»
   - Кнопка: «Unlock full blueprint — €5»
   - Вспомогательный текст: «See what could break — before it does.»
7. Пользователь нажимает кнопку → `isUnlocked = true`.
8. Состояние после разблокировки показывает:
   - Чего вы избегаете (из `blueprint.hiddenPain`)
   - Что может сломаться (из `blueprint.skepticView.whatCouldBreak`)
   - Компонент `ExecutionPressure` с 24-часовой кнопкой обязательства

## 5. Что НЕ включено (пока)

- Никакой интеграции со Stripe.
- Никакой серверной валидации платежей.
- Никакой учётной записи пользователя или сессии, привязанной к состоянию разблокировки.
- Никаких аналитических событий на нажатие разблокировки — пока измерять вручную.
- Состояние разблокировки сбрасывается при перезагрузке страницы — намеренно для этой фазы тестирования.

## 6. Правила тона

- Текст пэйволла прямой, не извиняющийся.
- «Want to see what's actually driving this?» — любопытство, не обвинение. Полный ответ уже дан; это действительно опциональная глубина.
- Никакого «upgrade to premium», никакого сравнения функций, никакого языка пробного периода.

## 7. Сигнал для наблюдения

Если пользователи последовательно нажимают «Unlock full blueprint — €5», ворота подтверждают:
- Пользователи получают реальную ценность от бесплатного ответа и хотят идти глубже.
- Фрейминг («hidden motivation map», «what could break») создаёт достаточно любопытства для нажатия.
- €5 — приемлемая цена за дополнительную глубину, а не просто за вердикт.

## 8. Следующие шаги (если сигнал положительный)

- Добавить реальный Stripe-чекаут за €5 единоразово.
- Или добавить подписку: неограниченные разблокировки за €X/месяц.
- Отслеживать частоту разблокировок на сессию и по типу решения.
