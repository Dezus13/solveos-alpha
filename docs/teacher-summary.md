# Teacher Summary

## What Problem SolveOS Solves

People often ask AI for advice but do not act on the answer. SolveOS solves this by turning advice into a decision, a next action, and a follow-up check.

## What The User Can Do

The user can enter a decision, receive a short verdict, see a suggested action, save the decision, and later record whether the action or outcome worked.

SolveOS also shows pending actions so the user is reminded to follow through.

## Technical Parts

SolveOS is built with Next.js, React, TypeScript, and local API routes.

The main technical parts are:

- A decision API that creates verdicts and action plans.
- A UI for entering decisions and reading results.
- A memory system that saves decision history.
- An action reminder system that tracks pending actions in the browser.
- A profile and identity system that changes when the user follows through or avoids action.
- Test scripts that check important routing and behavior.
