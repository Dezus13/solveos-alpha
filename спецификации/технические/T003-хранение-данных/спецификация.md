# T003 — Хранение данных

## Текущая архитектура

SolveOS использует несколько видов хранения.

Серверная история решений хранится через `lib/memory.ts`. В разработке это локальный файл `data/decisions.json`. В Vercel или похожем окружении запись в файл отключается; при наличии переменных окружения используется удаленное KV-хранилище.

Клиентские состояния хранятся в `localStorage`: история чата, действия, настройки и профиль.

## Важные файлы

- `lib/memory.ts`
- `lib/savedDecisions.ts`
- `lib/actionReminders.ts`
- `lib/userProfile.ts`
- `lib/settings.ts`
- `lib/settingsStore.ts`
- `app/api/memory/route.ts`
- `app/api/outcomes/route.ts`
- `data/decisions.json`

## Поток данных

1. Ответ `/api/solve` сохраняет решение в память.
2. Клиент хранит последние сообщения чата в `solveos.conversation.v1`.
3. Действия хранятся в `solveos_action_pressure_v1`.
4. Профиль хранится в `solveos_user_profile`.
5. Настройки внешнего вида хранятся в `solveos_settings`.
6. Журнал и панель читают эти данные и показывают состояние.

## Риски

- localStorage доступен только в браузере
- поврежденный JSON может сломать чтение, если не обрабатывать ошибки
- изменение ключей приведет к потере старых данных пользователя
- серверная память в локальном файле не подходит как долгосрочная база для продакшена

## Ограничения

- все обращения к `window` и `localStorage` должны быть клиентскими
- новые поля нужно добавлять совместимо со старыми записями
- удаленное хранилище должно быть явно настроено переменными окружения
- история чата не равна журналу решений

## Что нельзя менять случайно

- `solveos.conversation.v1`
- `solveos_action_pressure_v1`
- `solveos_user_profile`
- `solveos_settings`
- `solveos.productSettings.v1`
- формат `data/decisions.json`
