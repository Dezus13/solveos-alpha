# F004 — Режим глубокого анализа

## Цель

Дать пользователю больше разбора для сложного решения, не перегружая основной ответ.

## Пользовательский поток

1. Пользователь получает основной ответ.
2. Открывает блок глубокого анализа.
3. Система делает дополнительный запрос к `/api/solve`.
4. Пользователь видит подробный разбор.

## Ожидаемое поведение

- глубокий анализ запускается только когда есть текущая тема
- во время загрузки показывается понятное состояние
- ошибка показывается внутри блока анализа
- основной чат остается рабочим
- язык интерфейса соответствует текущему языку пользователя, где это поддержано

## Что не должно происходить

- глубокий анализ не должен заменять основной ответ
- блок не должен запускать бесконечные запросы
- простой запрос не должен автоматически превращаться в длинный отчет
- ошибка глубокого анализа не должна очищать текущий чат

## Критерии приемки

- пользователь может открыть и закрыть глубокий анализ
- загрузка и ошибка отображаются без падения страницы
- основной ответ остается видимым
- повторный запуск не дублирует хаотичные состояния
