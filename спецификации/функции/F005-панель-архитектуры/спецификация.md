# F005 — Панель архитектуры

## Родительская власть

Главнее этой спецификации: `спецификации/бизнес-контекст.md`.

Эта спецификация описывает вспомогательную панель и не может делать ее главным продуктовым опытом.

## Цель

Показать основателю и разработчику, какие внутренние части SolveOS участвуют в ответе: память, профиль, действия, уверенность и состояние системы.

## Границы

Внутри: показ состояния памяти, действий, профиля и понятных системных показателей.

Вне: изменение ответа модели, новые API-правила и новые источники данных.

## Пользовательский поток

1. Пользователь работает с главным экраном.
2. Боковая или вспомогательная панель показывает состояние системы.
3. После новых решений, действий или изменений профиля данные обновляются.

## Ожидаемое поведение

- панель помогает понять продукт, но не мешает главному чату
- данные берутся из существующих источников памяти, действий и настроек
- пустые состояния должны выглядеть спокойно
- метрики должны быть понятными и не обещать больше, чем продукт знает

## Что не должно происходить

- панель не должна становиться главным интерфейсом
- нельзя показывать внутренние промпты или скрытые инструкции модели
- нельзя добавлять декоративную сложность вместо понятных данных
- изменение панели не должно менять ответ API

## Что эта спецификация не имеет права определять

- бизнес-позиционирование
- контракт `/api/solve`
- схему хранения данных
- обязательные требования к user stories

## Критерии приемки

- панель рендерится без данных
- после решения отображаются обновленные показатели
- ошибки чтения localStorage не ломают страницу
- панель не меняет бизнес-логику ответов
