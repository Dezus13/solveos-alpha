# F001 — Чат и ответы

## Родительская власть

Главнее этой спецификации: `спецификации/бизнес-контекст.md`.

Эта спецификация описывает функциональное поведение чата и не может менять границы продукта.

## Цель

Дать пользователю место, где он вводит решение или уточнение, а SolveOS отвечает ясно и по делу.

## Границы

Внутри: ввод пользователя, текущий чат, короткие уточнения, показ ошибок ответа.

Вне: память решений, глубокий анализ, внутренняя логика модели и хранение данных.

## Пользовательский поток

1. Пользователь открывает главную страницу.
2. Вводит ситуацию или вопрос.
3. Нажимает отправку.
4. Видит ответ SolveOS.
5. Может задать короткое уточнение по той же теме.

## Ожидаемое поведение

- пустой ввод не отправляется
- непустой ввод принимается без искусственного минимума символов
- последние сообщения чата используются как контекст
- короткие уточнения вроде «почему?» должны пониматься из текущей темы
- ответ должен быть практичным, коротким и не похожим на обычную болтовню
- ошибка API должна показываться как понятное сообщение, не ломая страницу

## Что не должно происходить

- чат не должен превращаться в лендинг
- ответ не должен быть общей мотивационной речью
- уточнение не должно запускать полностью новую тему, если контекст ясен
- интерфейс не должен менять поведение сайта без отдельной задачи

## Что эта спецификация не имеет права определять

- бизнес-позиционирование SolveOS
- структуру `/api/solve`
- правила хранения истории
- поведение глубокого анализа

## Критерии приемки

- пользователь может отправить первый запрос
- пользователь может продолжить тему коротким уточнением
- история текущего чата сохраняется в браузере
- при ошибке пользователь видит понятный сбой
- runtime-код не меняется при правке этой спецификации
