# F002 — Память и история

## Цель

Сохранять прошлые решения и использовать их как контекст, чтобы SolveOS становился полезнее для повторного пользователя.

## Пользовательский поток

1. Пользователь получает ответ на решение.
2. Решение сохраняется в историю.
3. При следующих запросах система может учитывать прошлые решения, действия и исходы.
4. Пользователь может открыть журнал и посмотреть сохраненные решения.

## Ожидаемое поведение

- история решений сохраняет вопрос, вердикт, риски, следующий шаг и оценку
- локальная история чата хранится отдельно от журнала решений
- память помогает ответу только когда это уместно
- поврежденные или пустые данные не должны ломать приложение
- в Vercel приложение не должно пытаться писать в локальный файл

## Что не должно происходить

- память не должна пересказывать пользователю все старые данные
- локальный чат не должен считаться постоянной серверной историей
- старые записи не должны блокировать новый запрос
- нельзя менять формат хранения без миграции или явного описания

## Критерии приемки

- новое решение появляется в истории
- повторный запрос может получить контекст из прошлых решений
- журнал не падает при пустой истории
- невалидный JSON обрабатывается безопасно
