# Инструкции для работы со спецификациями

## Обязательный порядок

Перед любой реализацией:

- прочитать `спецификации/бизнес-контекст.md`
- определить связанные спецификации функций
- если задача пересекает границы продукта, прочитать связанные технические спецификации
- сравнить запрос пользователя с существующими спецификациями
- только после этого реализовывать, редактировать документацию или менять требования

## Как писать спецификации

- писать по-русски
- использовать простые слова
- описывать текущее поведение отдельно от будущих идей
- не добавлять сложность ради вида
- не дублировать старые документы целиком
- если старый документ полезен, кратко перенести смысл

## Где что лежит

- `бизнес-контекст.md` — зачем существует SolveOS
- `функции/` — пользовательские возможности
- `технические/` — устройство приложения и ограничения
- `истории-пользователя/` — практические сценарии
